### 2.2 配置 pom.xml

**关键依赖层级**:

1. **Spring Boot 父工程**: `spring-boot-starter-parent 3.2.5`
2. **依赖管理**: 通过 `langchain4j-bom` 统一管理版本号
3. **核心依赖**:

```xml
<!-- 使用 BOM 统一管理版本 -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-bom</artifactId>
            <version>${langchain4j.version}</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

**注意**: 这里使用了 `langchain4j-bom` 配合 `<scope>import</scope>`，这样所有 langchain4j 子模块就无需再单独指定版本号，防止版本冲突。

## 3. 配置层实现

### 3.1 application.yml 配置

配置文件分为几大区块：

```
server.port=8080
    ↓
spring:
  datasource → H2 文件数据库 (持久化)
  jpa → ddl-auto: update (自动建表)
  thymeleaf → 模板引擎配置
    ↓
assistant: (自定义配置)
  default-model → ollama / deepseek
  ollama → base-url, model-name, temperature...
  deepseek → base-url, api-key, model-name...
  mcp → servers 列表
  rag → 文档目录、检索参数
  agent → system-prompt, memory 配置
```

### 3.2 AssistantProperties.java

这是**配置属性映射类**，将 `application.yml` 中的 `assistant.*` 配置映射为 Java 对象：

```java
@ConfigurationProperties(prefix = "assistant")
public class AssistantProperties {
    private String defaultModel;
    private Ollama ollama;
    private Deepseek deepseek;
    private Mcp mcp;
    private Rag rag;
    private Agent agent;
    // ... 嵌套静态内部类
}
```

**为什么这样设计**: 使用 `@ConfigurationProperties` 可以集中管理所有自定义配置，比分散的 `@Value` 注解更易维护。

## 4. 数据模型层

### 4.1 JPA 实体设计

采用**一对多**关系：一个 `Conversation` 包含多个 `ChatMessage`

```
Conversation (对话会话)
  ├── id (主键)
  ├── title (标题，自动从首条消息生成)
  ├── modelType (使用的模型: ollama/deepseek)
  ├── summary (对话摘要，用于长期记忆)
  ├── status (ACTIVE / ARCHIVED)
  ├── createdAt / updatedAt
  └── messages (一对多 → ChatMessage)

ChatMessage (聊天消息)
  ├── id (主键)
  ├── conversation (多对一 → Conversation)
  ├── role (USER / ASSISTANT / SYSTEM / TOOL)
  ├── content (消息内容)
  ├── messageType (text / tool_call / tool_result)
  ├── toolInfo (Tool 调用信息 JSON)
  └── createdAt
```

### 4.2 Repository 层

```java
// 基础 CRUD 由 JpaRepository 提供
// 自定义查询:
public interface ConversationRepository extends JpaRepository<Conversation, Long> {
    List<Conversation> findByStatusOrderByUpdatedAtDesc(String status);
    
    @Query("SELECT c FROM Conversation c WHERE c.status = 'ACTIVE' ORDER BY c.updatedAt DESC")
    List<Conversation> findRecentActiveConversations();
}
```

**H2 数据库特点**: 嵌入式文件数据库 `jdbc:h2:file:./data/assistant-db`，数据持久化到本地文件，无需安装任何数据库服务，非常适合桌面应用场景。

## 5. LangChain4j 集成

### 5.1 核心配置 (LangChain4jConfig.java)

这是整个项目的**大脑**，负责所有 AI 相关 Bean 的初始化。

#### 5.1.1 模型配置流程

```
OllamaChatModel (本地)
   └── @Bean("ollamaChatModel") → ChatModel
   └── @Bean("ollamaStreamingChatModel") → StreamingChatModel

OpenAiChatModel (DeepSeek)
   └── @Bean("deepseekChatModel") → ChatModel
   └── @Bean("deepseekStreamingChatModel") → StreamingChatModel
```

**注意**: LangChain4j 1.x 版本中，`ChatLanguageModel` 已重命名为 `ChatModel`，`StreamingChatLanguageModel` 已重命名为 `StreamingChatModel`。

#### 5.1.2 模型选择策略 (ModelSelectionConfig.java)

```java
// 根据 application.yml 中的 default-model 自动切换
@Primary
public ChatModel chatModel(
    @Qualifier("ollamaChatModel") ChatModel ollamaModel,
    @Qualifier("deepseekChatModel") ChatModel deepseekModel) {
    
    return switch (defaultModel) {
        case "deepseek" -> deepseekModel;
        case "ollama"  -> ollamaModel;
        default        -> ollamaModel; // 默认回退
    };
}
```

#### 5.1.3 Embedding 模型配置

```java
@Bean
public EmbeddingModel embeddingModel() {
    // AllMiniLmL6V2 - 本地 ONNX 模型
    // 模型文件自动从 Maven 依赖加载，无需下载
    return new AllMiniLmL6V2EmbeddingModel();
}
```

### 5.2 AI Service 接口定义

这是 LangChain4j 的核心模式——**声明式 AI 服务**：

```java
public interface AssistantAiService {
    @SystemMessage("{{systemPrompt}}")
    String chat(
        @MemoryId Long conversationId,
        @UserMessage String userMessage,
        @V("systemPrompt") String systemPrompt
    );
}
```

**注解含义**:
- `@SystemMessage`: 系统提示词，定义 AI 角色
- `@MemoryId`: 指定会话 ID，LangChain4j 自动关联聊天记忆
- `@UserMessage`: 用户输入
- `@V("variableName")`: 注入动态变量

### 5.3 AiServices 构建

```java
AiServices.builder(AssistantAiService.class)
    .chatModel(chatModel)              // 注入 ChatModel
    .chatMemoryProvider(provider)       // 注入消息窗口记忆
    .tools(tools)                       // 注入工具
    .build();
```

**关键**: `chatMemoryProvider` 使用 `MessageWindowChatMemory`，它会自动维护一个滑动窗口的消息历史，在每次调用时将最近 N 条消息作为上下文传给 LLM。

## 6. 服务层实现

### 6.1 MemoryService（记忆服务）

**职责**: 管理对话的生命周期，持久化消息到数据库。

```
createConversation() → 新建对话
    ↓
saveUserMessage() → 持久化用户消息 + 自动生成标题
    ↓
saveAssistantMessage() → 持久化 AI 回复
    ↓
getConversationMessages() → 加载历史消息
    ↓
updateConversationSummary() → 更新对话摘要（长期记忆）
```

**自动标题生成逻辑**: 当对话标题为默认的"新对话"时，取首条用户消息的前 20 个字作为标题。

### 6.2 ChatService（核心聊天服务）

**聊天流程**:

```
用户输入 → Controller
    ↓
ChatService.chat(conversationId, message)
    ├── 1. 保存用户消息到数据库
    ├── 2. 构建 System Prompt（含系统提示 + 历史摘要）
    ├── 3. 调用 aiService.chat()
    │      ├── LangChain4j 自动注入聊天记忆（历史消息）
    │      ├── 模型判断是否需要调用 Tool
    │      ├── 如有需要，执行 Tool → 返回结果给模型
    │      └── 模型生成最终回复
    ├── 4. 保存 AI 回复到数据库
    └── 5. 返回 ChatResponse
```

**Tool 调用流程** (由 LangChain4j 自动处理):

```
用户: "现在几点？"
    ↓
LLM 分析 → 需要调用 getCurrentDateTime 工具
    ↓
LangChain4j 拦截 → 执行 AssistantTools.getCurrentDateTime()
    ↓
返回结果 "2026-05-19 20:30:00"
    ↓
LLM 根据结果生成自然语言回复: "现在是 2026年5月19日 20点30分。"
```

### 6.3 RagService（检索增强生成）

**RAG 流程**:

```
启动时: 
  loadDocumentsFromDirectory()
    ├── 扫描 data/documents/ 目录下的 .txt/.md/.json/.xml 文件
    ├── 读取文件内容
    ├── 用 EmbeddingModel 向量化
    └── 存入 InMemoryEmbeddingStore

查询时:
  search(query)
    ├── 1. 向量化用户查询
    ├── 2. 构建 EmbeddingSearchRequest
    ├── 3. 在向量库中搜索相似内容
    └── 4. 返回匹配文本作为 LLM 上下文
```

**Embedding 模型**: 使用 `AllMiniLmL6V2EmbeddingModel`（基于 ONNX 运行时），完全本地运行，无需联网，生成 384 维向量。

## 7. 工具与 MCP

### 7.1 AssistantTools（工具集合）

使用 `@Tool` 注解标记，LangChain4j 自动注册为 LLM 可调用的函数：

```java
@Component
public class AssistantTools {
    
    @Tool("获取当前的日期和时间")
    public String getCurrentDateTime() {
        return LocalDateTime.now().format(...);
    }
    
    @Tool("执行数学计算")
    public String calculate(@P("数学表达式") String expression) {
        // 使用 ScriptEngine 执行计算
    }
    
    @Tool("在本地知识库中搜索相关信息")
    public String searchKnowledgeBase(@P("搜索关键词") String query) {
        return ragService.search(query);
    }
}
```

**工作原理**: 
1. LLM 生成函数调用请求 → 2. LangChain4j 框架拦截 → 3. 通过反射调用对应 Java 方法 → 4. 返回结果给 LLM → 5. LLM 生成自然语言回复

### 7.2 MCP 集成

**MCP 服务框架**:

```
application.yml 配置:
  assistant.mcp.servers:
    - name: filesystem
      transport: stdio
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem", "./data"]

→ McpConfig 启动时连接 MCP 服务器
→ 预留 client 连接，未来可扩展
→ 提供 /api/status 接口查看 MCP 连接状态
```

## 8. Web 控制器

### 8.1 路由设计

```
页面路由:
  GET  /             → 聊天主页 (Thymeleaf 模板)

REST API:
  POST   /api/conversations              → 创建新对话
  GET    /api/conversations              → 获取对话列表
  GET    /api/conversations/{id}/messages → 获取消息历史
  POST   /api/chat                       → 发送消息
  DELETE /api/conversations/{id}         → 删除对话
  GET    /api/health                     → 健康检查
  GET    /api/status                     → 助手状态
```

### 8.2 聊天 API 请求示例

```bash
# 创建新对话
curl -X POST http://localhost:8080/api/conversations
→ {"conversationId": 1}

# 发送消息
curl -X POST http://localhost:8080/api/chat \
  -d "conversationId=1&message=你好"
→ {"conversationId": 1, "message": "你好！我是小智，有什么可以帮你的？"}
```

## 10. 编译验证与运行

### 10.1 依赖下载

```bash
cd desktop-assistant
mvn dependency:resolve
```

Maven 会自动从阿里云镜像下载所有依赖，其中几个关键的大文件：
- `onnxruntime-1.20.0.jar` (93MB) — ONNX 运行时
- `langchain4j-embeddings-all-minilm-l6-v2-1.14.1-beta24.jar` (83MB) — Embedding 模型文件

### 10.2 编译

```bash
mvn clean compile
```

**常见编译错误及解决**:

| 错误 | 原因 | 解决 |
|------|------|------|
| `找不到符号 ChatLanguageModel` | API 名称变更 | 改为 `ChatModel` |
| `找不到符号 findRelevant` | API 签名变更 | 改为 `search(EmbeddingSearchRequest)` |
| `embed() 返回类型不兼容` | 1.x 返回 `Response<Embedding>` | 调用 `.content()` 获取 Embedding |
| `@Builder.Default` 警告 | Lombok 默认值问题 | 显式添加 `@NoArgsConstructor @AllArgsConstructor` |

### 10.3 打包

```bash
mvn package -DskipTests
```

生成 `target/desktop-assistant-1.0.0.jar`，可直接 `java -jar` 运行。

### 10.4 运行

```bash
# 方式一：Maven 直接运行
mvn spring-boot:run

# 方式二：打包后运行
java -jar target/desktop-assistant-1.0.0.jar

# 方式三：指定 DeepSeek Key
set DEEPSEEK_API_KEY=sk-your-key
mvn spring-boot:run
```

## 附：关键 API 差异备忘

**LangChain4j 0.x → 1.x 迁移注意**:

| 0.x (旧) | 1.x (新) | 说明 |
|-----------|----------|------|
| `ChatLanguageModel` | `ChatModel` | 核心接口更名 |
| `StreamingChatLanguageModel` | `StreamingChatModel` | 流式接口更名 |
| `.chatLanguageModel(model)` | `.chatModel(model)` | AiServices builder 方法更名 |
| `embeddingStore.findRelevant(emb, k, minScore)` | `embeddingStore.search(EmbeddingSearchRequest)` | 搜索 API 变更 |
| `embeddingModel.embed(text).content()` | 同左 | 但返回类型变为 `Response<Embedding>` |
| `langchain4j-embeddings-all-minilm-lm-v2` | `langchain4j-embeddings-all-minilm-l6-v2` | 模块名修正（lm→l6） |
